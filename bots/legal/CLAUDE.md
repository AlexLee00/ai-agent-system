## 팀 개요

마스터가 법원 SW 감정인으로 활동 중. 저스틴팀이 감정 프로세스 전 단계를 자동화하여 초안을 제공하고, 마스터가 최종 검토·서명 후 법원에 제출.

## 핵심 워크플로우 (13단계)

```
1. 감정 촉탁 수신 → DB INSERT
2. 브리핑: 사건/감정소요 분석
2.5. 클레임+디펜스 (병렬): 양측 자료 분석
3. 가람+아틀라스 (병렬): 판례 분석
4. 렌즈: 소스코드 비교 분석
5. 감정착수계획서 작성 (마스터 검토)
6. 1차 질의서 발송 (마스터 검토)
7. 1차 현장확인 인터뷰
8. 2차 질의서 발송 (마스터 검토)
9. 2차 현장확인 인터뷰
10. 현장실사계획서 발송 (마스터 검토)
11. 현장실사: SW 기능 3단계 분류 (가동/부분가동/불가동)
12. 퀼+밸런스: 감정보고서 초안 → 저스틴 최종 검토
13. 마스터 서명 → 법원 제출 → RAG 저장
```

## 디렉토리 구조

```
bots/legal/
├── lib/          ← 에이전트 + 유틸리티
├── scripts/      ← CLI 스크립트
├── templates/    ← 감정서 템플릿
├── cases/        ← 사건 데이터 (gitignore!)
├── context/      ← 페르소나, 가이드라인, 법률 용어
└── migrations/   ← DB 스키마
```

## 보안 원칙

- `cases/` 디렉토리: GitHub push 절대 금지 (소스코드 보안)
- 감정서는 에이전트가 초안만 작성, 마스터가 최종 서명
- 중립성: 원고/피고 어느 쪽에도 편향 금지
