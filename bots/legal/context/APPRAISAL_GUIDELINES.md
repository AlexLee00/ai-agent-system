# 감정 가이드라인 — 법원 SW 감정 작성 원칙

## 1. 감정서 기본 구조 (법원 양식)

```
제 목: 감 정 서
사건번호: [법원명] [연도]가합[번호]
사  건: [원고] vs [피고]
감정인: [마스터 성명]

1. 감정 개요
   1.1 감정 목적
   1.2 감정 범위 및 방법
   1.3 감정 기간

2. 사건 및 감정소요 분석
   2.1 사건 경위
   2.2 쟁점 정리
   2.3 감정 착수 결정 사항

3. 분석 방법론
   3.1 사용 도구 및 기술
   3.2 분석 기준

4. 분석 결과
   4.1 [감정 유형별 세부 결과]

5. 판례 참조
   5.1 국내 판례
   5.2 해외 판례

6. 현장실사 결과 (해당 시)
   6.1 SW 기능 분류표
   6.2 가동 판정표

7. 감정 의견
   7.1 결론
   7.2 근거

8. 첨부 자료
```

## 2. 감정 유형별 특화 지침

### 저작권 침해 (copyright)
- 원고 코드와 피고 코드의 유사도 수치 제시 (%)
- 창작성 있는 표현의 유사성 여부 (아이디어/표현 이분법 적용)
- 독자 개발 가능성 분석 (오픈소스, 공통 라이브러리 등)
- 접근 가능성 여부 (원고 코드에 피고가 접근할 수 있었는가)

### 소프트웨어 하자 (defect)
- 계약 명세 vs 실제 납품물 비교
- 기능별 가동/부분가동/불가동 판정
- 하자의 중요성 등급 (치명적/중요/경미)
- 하자 보완 가능성 및 예상 비용

### 계약 위반 (contract)
- 계약서 원문 기준 의무 이행 여부
- 지연/미이행 사항 목록화
- 계약상 손해배상 산정 근거

### 영업비밀 침해 (trade_secret)
- 비밀관리성 요건 검토
- 기술적 유용성 요건 검토
- 침해 방법 및 경로 분석

## 3. SW 기능 3단계 분류 체계

```
1스텝 (대분류): 모듈/시스템 단위
  예: 회원관리, 결제, 재고관리, 게시판

2스텝 (중분류): 기능 단위
  예: 회원가입, 로그인, 비밀번호 변경

3스텝 (소분류): 세부 기능/화면 단위
  예: 이메일 인증, 소셜 로그인, 휴대폰 인증
```

## 4. 가동 판정 기준

| 판정 | 기준 | 표기 |
|------|------|------|
| 가동 | 해당 기능이 정상 작동 | ✅ 가동 |
| 부분가동 | 일부 기능만 작동, 나머지 오류/미구현 | ⚠️ 부분가동 |
| 불가동 | 해당 기능 전체가 작동하지 않음 | ❌ 불가동 |

## 5. 중립성 원칙

- 감정서에 "~으로 보인다" "~할 것이다" 등 추측성 표현 최소화
- "확인되었다" "측정되었다" 등 객관적 표현 사용
- 원고/피고 편향적 표현 금지
- 불확실한 사항은 "추가 확인이 필요하다" 명시
