## ⛔ 절대 규칙 (위반 금지)

> **모든 개발·테스트는 DEV 모드에서만 진행한다.**
> **OPS 모드 전환은 반드시 사용자와 협의 후에만 허용한다.**

| 상황 | 클로드의 행동 |
|------|-------------|
| 기능 추가·수정·버그픽스 | DEV 모드 (`dry_run=true`) 에서만 개발 및 테스트 |
| 드라이런 테스트 통과 | 사용자에게 결과 보고 → **OPS 전환 여부는 사용자가 결정** |
| secrets.json 변경 요청 | `dry_run: false` 변경은 사용자 명시적 지시 없이 절대 불가 |
| `start-invest-ops.sh` 실행 | 사용자가 직접 실행하거나 명시적으로 허가한 경우에만 |
| launchd OPS 등록 | 사용자 협의 후에만 (`launchctl load ai.invest.pipeline.plist`) |

**클로드는 OPS 배포를 먼저 제안하거나 자동으로 수행하지 않는다.**

## 텔레그램 명령어

| 명령 | 동작 |
|------|------|
| `/포지션` | 현재 오픈 포지션 조회 |
| `/잔고` | 바이낸스 + 업비트 잔고 |
| `/신호` | 최근 신호 목록 |
| `/수익` | 오늘 실현 수익 |
| `/분석 BTC/USDT` | 즉시 TA 분석 실행 |
| `/드라이런 BUY BTC/USDT 100` | 드라이런 주문 테스트 |

## 주의사항

1. **실거래 전**: secrets.json에 API 키 설정 + `dry_run: false` 변경
2. **바이낸스 API 보안**: IP 화이트리스트 + 출금 권한 비활성화
3. **업비트 출금**: 별도 키 분리 필수
4. **손절 규칙**: 코드 변경 없이 임의 무시 금지

## 드라이런 확인
