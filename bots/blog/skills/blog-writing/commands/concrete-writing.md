# Concrete Writing

- Start with the reader's concrete situation before abstract claims.
- Replace generic claims with one operating example, one tradeoff, and one action.
- Use BLs1 crank lessons only as direction; do not paste them verbatim.
- For IT posts, include selection criteria, limits, and implementation cost.
- For book reviews, include reading moment, personal application, and changed view.
- Keep `genre:it` learnings out of book drafts and `genre:book` learnings out of IT drafts.
- If the draft smells like a repeating title template, force a new question or experience frame.
