# Comment Strategy

- Use comment evolver output as shadow proposal only.
- Posted-success events are evidence; failed post attempts are not positive evidence.
- Keep classifier strategy unchanged unless a later master-approved spec says otherwise.
- For high `기타` share, propose taxonomy cleanup before changing posting behavior.
- For low performance, propose tone/source changes, not higher volume by default.
- Never trigger real comments from A2A, skill docs, or weekly shadow reports.
