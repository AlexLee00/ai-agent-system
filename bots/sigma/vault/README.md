# Sigma PARA Vault

Markdown second-brain vault for Sigma.

- `00-inbox`: raw capture zone.
- `10-projects`: active work with clear outcomes.
- `20-areas`: ongoing team operating areas.
- `30-resources`: reusable research, patterns, references.
- `40-archives`: completed or retired material.

File moves are dry-run by default through `runtime-sigma-vault-classifier.ts`; use `--apply` only when classification results are reviewed.
