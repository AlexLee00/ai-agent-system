## 4. 마스터 선호 방식

### 4-1. 단계별 롤백 포인트

마스터는 **모든 큰 작업 전 `pre: ...롤백 포인트` 커밋** 습관.
- 실수 시 `git reset --hard <sha>`로 30초 복구
- 메티/코덱스는 이 패턴 존중

### 4-2. 한 번에 일괄 처리 선호

- ✅ "옵션 D 전체 진행 + 전수조사 + 다윈팀까지 예고"
- ❌ "단계마다 물어보기"

단, **프로덕션 영향 작업**은 반드시 승인 받기.

### 4-3. 기술 스택 변경에 보수적

- 이미 쓰는 것 최대 재활용 (워커 피드백 893줄 에디팀 재사용 등)

- 예: `jido_memory` 미존재 → agent-memory.ts 수동 포팅 선택

### 4-4. 문서 체계 엄격

- 설계·전략·개발·히스토리·학술·코덱스·가이드 7대 카테고리
