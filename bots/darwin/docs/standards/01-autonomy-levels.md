# Darwin V2 — 자율성 레벨 정의

> 최종 업데이트: 2026-04-18

---

## 개요

Darwin V2는 3단계 자율성 레벨(L3/L4/L5)을 운영합니다. 레벨이 높을수록 사람 개입 없이 더 많은 작업을 자동으로 수행합니다. 시스템 기본 레벨은 L3로 시작하지만, **현재 live 운영 레벨은 L5**입니다.

---

## 레벨 정의

### L3 — 경로 오류 수정 (현재 레벨)

**설명**: 파이프라인은 완전히 실행되지만, Edison(구현 에이전트)의 모든 실행에 수동 승인이 필요합니다.

**동작**:
- 논문 스캔, 평가, 계획 수립: 자동 실행
- Edison 코드 생성: 자동 실행
- **코드 적용(apply)**: 마스터 명시 승인 필수
- 검증 실패 시: 마스터에게 알림 후 중단

**환경변수**:
```
DARWIN_V2_ENABLED=true
DARWIN_TIER2_AUTO_APPLY=false
```

---

### L4 — 반자동 (Semi-Auto)

**설명**: Edison이 승인 없이 자동으로 실행됩니다. 단, 코드 적용은 여전히 마스터 승인이 필요합니다.

**동작**:
- 논문 스캔 → 평가 → 계획 → Edison 실행: 전부 자동
- **코드 적용(apply)**: 마스터 승인 필요
- 검증 실패 시: 자동 롤백 + 마스터 알림

**업그레이드 기준** (L3 → L4):
- 연속 성공 횟수: **5회 이상**
- 운영 기간: **7일 이상**
- 마스터 명시 승인

---

### L5 — 완전 자동 (Full-Auto)

**설명**: 전체 파이프라인(스캔 → 평가 → 구현 → 검증 → main 적용)이 완전 자율로 실행됩니다.

**동작**:
- 모든 단계 자동 실행
- **코드 적용(apply)**: 자동으로 main에 적용
- 검증 실패 시: 자동 롤백 + 마스터 알림

**업그레이드 기준** (L4 → L5):
- 연속 성공 횟수: **10회 이상**
- 실제 적용 사례: **3건 이상**
- 운영 기간: **14일 이상**
- 마스터 명시 승인

**환경변수**:
```
DARWIN_V2_ENABLED=true
DARWIN_TIER2_AUTO_APPLY=true
```

---

## 자동 다운그레이드 규칙

| 이벤트 | 동작 |
|--------|------|
| 파이프라인 실패 1회 | `consecutive_successes` 초기화 (0으로 리셋) |
| 검증 실패 | L5인 경우 L4로 즉시 다운그레이드 |
| Kill Switch 활성화 | 모든 레벨에서 즉시 중단 |
| 일일 비용 상한 초과 | 당일 파이프라인 중단, 마스터 알림 |

---

## Kill Switch

Kill Switch는 모든 자율성 레벨을 즉시 무효화합니다.

```bash
# Kill Switch 활성화 (즉시 모든 자동 실행 중단)
DARWIN_KILL_SWITCH=true

# Kill Switch 해제 (마스터 명시 승인 후)
DARWIN_KILL_SWITCH=false
```

Kill Switch 활성화 시:
- 진행 중인 파이프라인: 현재 단계 완료 후 중단
- 새 파이프라인: 시작 불가
- JayBus에 `kill_switch_activated` 이벤트 발행

## 현재 live 운영 기준 (2026-04-20)

- `DARWIN_AUTONOMY_LEVEL=5`
- `DARWIN_L5_ENABLED=true`
- `DARWIN_KILL_SWITCH=false`
- `DARWIN_TIER2_AUTO_APPLY=true`
- `DARWIN_SHADOW_MODE=false`

정상 성공 경로:
- 구현 승인 버튼 없음
- 적용 승인 버튼 없음
- 공용 `postAlarm` 알림 경로 사용

예외 경로:
- 실패
- 충돌
- 수동 검토 필요

위 케이스만 텔레그램 수동 개입 버튼을 허용한다.
