# 제이 라우팅 계획

최종 업데이트: 2026-03-14

## 원칙

- API 모델은 기본 응답 경로에서 최대한 제외한다.
- 자연어는 유지하되, 명확한 운영 기능은 direct routing으로 처리한다.
- 복잡한 코드/운영 분석은 Codex 계정 경로로 분리한다.
- 모호한 자유 질의만 마지막에 LLM fallback으로 남긴다.

## Direct Routing 우선 대상

다음 기능은 자연어로 받아도 LLM 없이 직접 실행하는 것이 맞다.

- 시스템 상태
- 비용 조회
- 알람 큐 조회
- 무음 설정/해제
- 야간 브리핑
- 시스템 안정성 현황
- 텔레그램 상태
- 속도 체크
- 루나 상태
- 루나 리포트
- 루나 거래 일시정지/재개
- 업비트 잔고
- 바이낸스 잔고
- 암호화폐 현재가
- KIS 국내주식 잔고
- KIS 해외주식 잔고
- 스카 예약 조회
- 스카 매출 조회
- 스카 알람 조회
- 앤디/지미 재시작
- 덱스터 기본 점검
- 덱스터 전체 점검
- 덱스터 자동 수정
- 덱스터 일일 보고
- 아처 실행

## Codex 계정 경로 대상

다음 종류는 일반 대화형 LLM보다 Codex 계정 기반 경로가 더 적합하다.

- 코드 수정 요청
- 에러 원인 심층 분석
- 설정 리팩터링 제안
- 대규모 파일 비교
- 성능 병목 추적
- 아키텍처 정리
- 테스트 추가/보강
- 로그를 기반으로 한 수정 패치 생성

## 현재 1차 적용

- `속도 체크`
- `모델 속도 테스트`
- `제일 빠른 모델 뭐야`
- `비용 얼마야`
- `토큰 사용량`
- `텔레그램 상태`
- `최근 오류 보여줘`
- `로그 확인`

위 표현은 direct routing으로 처리한다.

## 다음 우선순위

1. `비용 체크` 결과를 더 짧은 운영 응답 형식으로 고정
2. `텔레그램 상태` direct health 응답 정교화
3. `루나/스카/클로드 로그 확인` direct routing 추가
4. `재시작` 계열을 팀별 direct routing으로 분리
5. Codex 계정 경로를 운영/코드 질의 전용 서브에이전트로 연결
