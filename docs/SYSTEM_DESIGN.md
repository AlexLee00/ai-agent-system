## 1. 시스템 개요

### 핵심 철학

> 사용자는 **메인 봇(팀장)에게만 지시**하고, 메인 봇이 알아서 팀원 봇에게 업무를 분배하고 결과를 취합하여 보고한다.

### 전체 구조도

```
👤 사용자
    │
    ▼ 지시 (자연어)
┌─────────────────────────────┐
│     🤖 메인 봇 (팀장)         │
│     - 지시 해석               │
│     - 업무 분배               │
│     - 결과 취합 & 보고         │
└──────────────┬──────────────┘
               │
    ┌──────────┼──────────────────────┐
    │          │          │           │           │
    ▼          ▼          ▼           ▼           ▼
📅 예약봇  🗓️ 비서봇  💼 업무봇  🎓 학술봇  ⚖️ 판례봇
```

## 6. n8n 워크플로우 설계

### 메인 워크플로우 (오케스트레이터)

```
[사용자 입력]
      ↓
[의도 분석] → 어떤 봇이 필요한가?
      ↓
[병렬 실행 분기]
  ├── 예약 관련? → 예약관리봇 호출
  ├── 일정 관련? → 개인비서봇 호출
  ├── 업무 관련? → 업무봇 호출
  ├── 논문 관련? → 학술봇 호출
  └── 판례 관련? → 판례봇 호출
      ↓
[결과 취합]
      ↓
[통합 보고 → 사용자]
```

### 예약관리 자동화 워크플로우

```
[10분 간격 트리거]
      ↓
[네이버 스마트플레이스 로그인]
      ↓
[홈 URL 직접 이동]  ← 이슈 해결 방향
      ↓
[예약 현황 파싱]
  ├── 확정 예약 수
  ├── 이용 중 예약 수
  └── 취소 예약 수
      ↓
[신규 예약 감지?]
  ├── Yes → 텔레그램 알림 발송
  └── No → 대기
      ↓
[데이터 저장 및 로그 기록]
```
