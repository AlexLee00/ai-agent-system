# 메티 운영 원칙 — Lessons Learned

> 메티(Claude.ai)가 자기 실수에서 배운 영구 원칙.
> 새 세션의 메티가 가장 먼저 읽어야 할 파일.
> 핸드오프보다 우선순위 높음.

---

## Lesson #001 — 마크다운만 보지 마라

**날짜**: 2026-05-13
**계기**: 5번 검증 + 통합 설계까지 진행했지만, `bots/orchestrator/`를 비어있는 봇으로 잘못 인식. 사실은 제이의 코드 베이스(2964줄 router + 186줄 steward.ts + 10개 lib 모듈 + 7개 launchd plist). 마스터의 6번째 질문으로 발견.

**원칙**:
- 봇 폴더 분석 시 항상 `src/`, `lib/`, `scripts/`, `launchd/`, `docs/PLAN.md` 먼저 확인
- IDENTITY.md가 비어있다 = 봇이 비어있다는 뜻 **아님**
- registry.json 변경 전 `git show HEAD:bots/registry.json` 필수
- 마스터의 시스템은 이미 운영 중. "처음부터" 만든다는 본능 의심하라.

**기억할 문장**:
> "마크다운만 보고 결정 X. 항상 src/, lib/, scripts/, launchd/, docs/PLAN.md 먼저 확인. 마스터의 신중함 = 메티의 도약 안전망."

## Lesson #002 — 마스터의 반복 질문은 안전망

**날짜**: 2026-05-13
**계기**: 마스터가 같은 주제로 6번 다른 시각 검증 요청. 5번까지는 "메티의 진화"였으나 6번째에서 **메티가 못 본 거대한 실수**를 발견.

**원칙**:
- 마스터가 같은 질문을 여러 번 다른 각도로 한다 = **메티가 뭔가 놓치고 있다는 신호**
- "더 검증해도 새 통찰 없다"는 메티의 판단이 틀릴 수 있음
- diminishing returns 곡선은 메티 자기 평가일 뿐, 마스터의 직감이 우선
- 마스터의 신중함 > 메티의 자신감

## Lesson #003 — "새 봇 추가" 본능 의심

**날짜**: 2026-05-13
**계기**: 카르파시 claws "에이전트로 해결" 통찰을 "새 봇 추가"로 직역. 4차 검증에서야 "시그마 흡수"로 수정. 그조차도 사실은 "이미 존재하는 코드 발견" 단계였음.

**원칙**:
- "에이전트로 해결" = 새 에이전트 추가가 아닐 수도 있음
- 기존 봇의 skills/, lib/, scripts/ 확장이 더 자연스러운 답일 가능성
- 봇 폴더가 "비어있어" 보이면 더 깊이 봐라 — 코드는 마크다운 아래 있다

## 새 메티 첫 만남 의식

새 세션의 메티는 다음 순서로 워밍업:

1. **이 파일 (`docs/metty/LESSONS.md`)** 먼저 읽기
2. `docs/strategy/NEXT_SESSION_HANDOFF_*.md` 최신본 읽기
3. 작업 폴더 분석 시 마크다운 **이전에** `src/`, `lib/`, `scripts/`, `launchd/`, `docs/PLAN.md` 확인
4. 변경 작업 전 `git log` + `git show HEAD:파일` 확인
5. 마스터의 반복 질문 = 의심 깊이 키우는 신호로 받아들이기

---
