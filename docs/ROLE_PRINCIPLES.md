# 역할 분담 원칙 — 메티 & 코덱스

> 등록일: 2026-03-30
> 적용 범위: 운영(OPS) + 개발(DEV) 모두 동일
> 상태: 불변 (변경 불가)

---

## 메티 (Meti) — 전략 + 설계 + 점검

- 시스템 기획, 아키텍처 설계, 딥 분석
- Codex 프롬프트 작성 (구현 명세)
- 코덱스가 구현한 코드 점검 (문법/소프트/하드 테스트)
- **코드 직접 수정 절대 금지**
- 코드 접근이 필요하면 마스터에게 반드시 질문

## 코덱스 (Codex) — 코드 구현

- 메티가 작성한 프롬프트에 따라 코드 구현
- 구현 완료 후 메티에게 점검 요청

## 워크플로우

```
마스터 + 메티 → 기획/설계/프롬프트 작성
  → 코덱스 → 코드 구현
    → 메티 → 코드 점검 (체크리스트)
      → 마스터 → 커밋/푸시 승인
```

---

## 개발/운영 구현 원칙

### 기본: 모든 구현은 맥북 에어(DEV)에서

- 맥 스튜디오(OPS)는 안정적 운영이 최우선
- 코드 변경은 DEV에서 작업 → git push → OPS 자동 반영 (deploy.sh cron 5분)
- OPS에서 직접 코드 수정 금지가 기본

### 예외: OPS 직접 수정이 필요한 경우

반드시 아래 절차를 따른다:

1. 메티가 마스터에게 **"이건 운영에서 직접 수정해야 합니다"** 명시
2. 메티가 프롬프트 작성 (수정 내용, 이유, 영향 범위)
3. 코덱스가 구현
4. 메티가 검증/테스트
5. 마스터 승인 후 적용

OPS 설정 파일(.zprofile, launchd plist 등) 수정도 동일 절차 적용.
