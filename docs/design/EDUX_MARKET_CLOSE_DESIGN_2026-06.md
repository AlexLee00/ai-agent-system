## 5. CODEX-EDUX-CLOSE 범위 (코덱스 1차 작업)

1. `runtime-edux-kis-daily.ts`: SLOT 인자 지원(0900|1600 — crypto 패턴 동일) + 1600 분기(마감 데이터 fetch + 휴장 가드 E-1)
2. `runtime-edux-overseas-daily.ts`: SLOT(2200|0630) + 0630 분기 (E-1 NYSE 가드)
3. `edux-formatter.ts`: buildKisCloseSystemPrompt/UserPrompt + buildOverseasCloseSystemPrompt/UserPrompt (§2 블록 명세 + §3 규칙: why-it-matters/확정치/시리즈 예고/면책 고정) + formatPost 슬롯 라우팅
4. E-4: 0900/2200 게시 시 metadata.watchPoints 저장 + 1600/0630에서 조회
5. plist 2개: `ai.edux.kis-daily-1600.plist`, `ai.edux.overseas-daily-0630.plist` (파일만 — 등록은 마스터)
6. 게이트 E-3 + 스모크: TS-EX-1(1600 dry-run 6블록+면책+plan-vs-actual 블록 존재) TS-EX-2(0630 동일) TS-EX-3(주말/휴장 -> skipped_holiday) TS-EX-4(기존 5슬롯 회귀 — dry-run 무변경) TS-EX-5(watchPoints 저장->회고 인용)
7. 안전: live 게시는 dry-run 검증 후 마스터 승인 — 기존 live gate 경로 준수. 기존 5슬롯 개선은 **마스터 승인(2026-06-12)으로 동시 진행** — 단 범위는 공통 포맷 규칙 주입+후처리 정규화+why-it-matters+시리즈 예고로 한정(6블록 구조 자체는 비변경), 전 7슬롯 dry-run 산출물을 마스터 확인 후 일괄 전환.

## 7. 부록 — 확정 포맷 규칙 + 예문 (2026-06-12 마스터 확정)

### 7.1 공통 포맷 규칙 (전 7슬롯, 코드 상수 EDUX_FORMAT_RULES로 프롬프트 주입)
1. 섹션(■) 사이: 빈 줄 **정확히 1개**
2. ■ 제목 다음 줄: **바로 내용** (제목 아래 빈 줄 금지)
3. 항목은 1줄 1항목 (모바일 가독 — 가로 나열 최소화, 동급 짧은 항목만 · 구분 허용)
4. 해석 줄(-> / 💡)은 수치 줄과 분리된 별도 줄
5. why-it-matters: 핵심 이슈에 "💡 왜 중요한가: ..." 1줄 의무
6. 시리즈 예고: 본문 끝에 다음 슬롯 1줄 예고 (사이클: crypto0600 -> US마감0630 -> 국내장전0900 -> crypto1400 -> 국내마감1600 -> NY장전2200 -> crypto2230 -> 익일 crypto0600)
7. 마지막 줄: 고정 면책(PUBLIC_MARKET_BRIEF_DISCLAIMER)
