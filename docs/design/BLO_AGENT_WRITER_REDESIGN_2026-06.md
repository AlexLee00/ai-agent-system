## 1. 비전 (마스터 확정)

**최종 목표: 스스로 성장하는 포스팅 작가 에이전트** — 주변 이웃을 확장하고 동적으로 관리하는 에이전트.
- 활동 범위: **포스팅 발행 + 블로그 댓글/공감 + Edu-X 게시물 발행** (이 3축 유지)
- 페이스북/인스타 자동게시 **삭제** (MCP 등 툴은 보존 — 차후 확장), 마케팅 로직 **off** (차후 확장)
- 강의는 '에이전트 입문' — 일반인 대상, codex/claude code 공식문서 기반, 에이전트로 업무 자동화하는
  아주 초보적인 내용 + 프롬프트 설정 등 꿀팁 다수
- 지난 강의·일반 포스팅과의 연계성: 업로드 본문을 시그마 대도서관(RAG)에 적재, 피드백 루프로 지속 학습
- 자기 댓글·이웃 댓글 학습 -> 유형별 동적 대응 / IT 인기 게시물 패턴 연구 -> 형식 리디자인

### 1-1. 보편 성장 루프 (마스터 확정 2026-06-13 — 시스템 핵심 패턴)
```
생성/발행 -> 마스터 예약발행 등록(익일 07~08시 실제 등록) -> 초안본+실제 등록본 DB 저장
 -> 피드백 학습(등록 차이 diff + 외부 IT 인기 패턴) -> 다음 생성에 적용 -> (무한 반복)
```
이 루프를 **포스팅뿐 아니라 댓글, 좋아요(공감), Edu-X 게시물 등록에도 동일 적용** — 모든 활동이
"실행 -> 실제 결과 관측 -> 차이 학습 -> 다음 실행 개선"의 성장 사이클을 갖는 에이전트 시스템.

## 3. B-Phase 상세 설계

### B1. 정체성·커리큘럼 (기반)
- 강의 시리즈 리브랜딩: 진행 중 4강을 **'에이전트 입문'으로 승계**(연속성 유지 — 권고안, 마스터 확인 대기)
- 커리큘럼 재설계: **codex/claude code 공식문서**(Anthropic docs, OpenAI codex docs) 기반.
  구성 원칙 — 일반인 눈높이: ① 도구 사용법(설치/설정/프롬프트 기본) ② 꿀팁(프롬프트 패턴, 자주 하는 실수)
  ③ 초보 에이전트 만들기(업무 자동화 미니 프로젝트). curriculum-planner.ts 재활용.
- 토픽 서칭 교체: "노드 최신정보" -> "커리큘럼 관련(codex/claude code/에이전트) 최신정보" + 유튜브 서칭(2차)
- 소셜 정리: 페북/인스타 plist 삭제(마스터), 관련 launchd 잡 제거, env 가드·MCP 툴 보존. 마케팅 로직 off 가드.
- CLAUDE.md 전면 갱신(비전·활동 3축·보류 항목 명문화) + blog.db 등 잔재 정리.

### B2. 시그마 대도서관 + 피드백 루프 (학습 기반)
- 신규 지식고: pgvector + Qwen3-Embedding(1024차원) 기반 — 시그마팀 RAG 자산 확인 후 통합/신설 결정.
- 적재 파이프라인 3종: ① blog.posts 본문 파싱 -> 청크 -> 임베딩(과거 120+4강 일괄 + 신규 게시 자동)
  ② 자기 댓글 + 이웃 댓글(comments/comment_actions — 데이터 기존재) ③ IT 인기 게시물(외부 서칭 수집)
- 소비: 포스팅 작성 시 RAG 컨텍스트 주입 — "지난 강의 N강에서 다룬 X" 연계가 가능한 작가.
- 피드백 루프: 게시 -> 적재 -> 다음 작성 참조 -> 성과(조회/댓글) 회수 -> 패턴 학습 (에러 피드백 루프 패턴 확장).
- **B2-4. 발행본 vs 최종본 diff 피드백 (마스터 확정 2026-06-13)**: 마스터가 블로그에서 직접 수정/삭제한
  최종 등록본을 발행 후 재파싱(기존 네이버 수집 자산 확장) -> DB 원본과 diff -> LLM 분석 -> 선호 패턴
  적재·학습. **기존 feedback-learner.ts(465줄) 재활용·부활** — 코드는 존재하나 master_feedback 입력 경로
  부재로 반쪽 동작 중. 신규: 최종본 수집기 + diff 분석 + 대도서관 적재 + 작성 프롬프트 반영.
  **타이밍(마스터 운영 실태)**: 발행본은 마스터가 예약발행으로 등록 -> **익일 07~08시 실제 등록** ->
  재파싱은 **익일 08:30 이후**(launchd) — 발행 직후가 아니라 실제 등록 후라야 마스터 수정분 포착.
  **루프 확장**: 동일 패턴을 댓글/공감(작성 댓글 vs 실제 노출·반응), Edu-X(게시본 vs 노출 성과)에
  단계적 적용 — B4/Edu-X 트랙에서 각각 구현.

### B3. 포스팅 형식 리디자인 (품질)
- 인기 IT 게시물 패턴 연구: 외부 서칭(에이전트 커뮤니티/레딧/깃헙 인기 레포/Anthropic 공식 깃헙·게시물,
  네이버 IT 인기글) -> 끌리는 제목/도입/구조/길이 패턴 추출 -> **대도서관 적재**(B2 소비).
- 형식 규칙 확정: Edu-X 검증 방식 — 패턴 연구 결과로 규칙 초안 -> **마스터 예문 확정** -> 규칙 상수 주입
  + 후처리 정규화(코드가 보장) + few-shot.
- 기존 humanize-agent/quality-checker와 연결(형식 규칙을 품질 게이트 항목으로).

### B4. 댓글 유형별 동적 대응 (이웃 관리)
- commenter.ts(6,215줄) 분할 리팩터와 한 작업으로: 코어(브라우저 액션) / 후보 선정 / **유형 분류기(신규)** /
  대응 전략(신규) 모듈 분리.
- 유형 분류: 이웃 댓글 -> 질문/감사/공감/스팸/제안 등 분류(hub LLM, 결정론 폴백) -> 유형별 응답 전략
  (답글 템포·깊이·이웃 방문 우선순위) -> 대도서관에서 과거 상호작용 참조.
- 이웃 확장·관리 동적화: 활동성·반응률 기반 이웃 우선순위(기존 후보 로직 강화).
- **B4-2. 댓글/공감 성장 루프 (§1-1 적용)**: 작성 댓글 초안 vs 실제 노출본 + 이웃 반응(답글/공감/재방문)
  회수 -> diff·반응 학습 적재 -> 유형별 대응 전략 자동 보정 -> 다음 댓글/공감 활동에 적용.
  공감(좋아요)은 대상 선택 -> 반응 회수 -> 우선순위 학습의 경량 루프.

### B5. 루나 패턴 이식 + 승급 체계 (구조)
- **사전 작업 B5-0 (마스터 지시 2026-06-13): 작성 에이전트 액션 로직 정밀 재검토** — blo.ts(2,697줄)/
  maestro.ts의 작성 플로우(준비->작가 선택->단계 실행->품질->후처리) 전수 추적, 단계별 적절성·중복·
  누락 평가 보고서 -> B4/B5 재설계의 입력.
- 작가 에이전트 분업(루나 A2A 패턴): 토픽 기획 / 작가 / 검수(humanize+quality) / 이웃 관리 역할 경계 명확화.
- shadow -> gate 승급: 신규 파이프라인(B1~B4 산출)은 **V2 live 무중단 병행 shadow** -> blog promotion gate
  (H6/GATE-R/edux-gate 표준 — contract+evidence+--apply 차단) -> 마스터 승인 live 전환.
- V3 부품 회수: Reddit 트렌드·Humanize 경로는 채택, 8채널 노출·C-Rank는 후순위 백로그.

### B6. Edu-X 성장 루프 (§1-1 적용 — 활동 3축 완성)
- Edu-X 게시본(7슬롯) vs 노출 성과(X 지표: 노출/참여, 가용 범위 실측 후 확정) 회수 -> 대도서관 적재
  -> 슬롯별 패턴 학습(어떤 구성·문구가 반응을 얻는가) -> 다음 게시 프롬프트 반영.
- 전제: 블로 B2 대도서관 가동 + Edu-X 마감 슬롯 live 전환(EDUX_TRACKER §B). 구현은 bots/edu-x 측이지만
  학습 루프 설계·지식고는 블로 트랙과 공유(동일 §1-1 패턴) — CODEX-B6.

## 6. 안전 원칙
- live 매일 2편 **무중단** — V2 경로 비파괴, 신규는 shadow 병행 후 gate 승급.
- LLM은 hub gateway 강제 유지(현행 정합 보존). 시그마 대도서관 적재 소스는 read-only.
- plist 삭제/등록·DDL·env·live 전환은 마스터. 코덱스는 파일만.
- 기존 안전 규칙(면책/투자권유 금지 등 Edu-X 연계 포함) 한 글자도 변경 금지 원칙 유지.
