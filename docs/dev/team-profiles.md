## 1. 클로드팀

**역할**: ai-agent-system 전체 소프트웨어 유지보수 전담
**위치**: `bots/claude/`
**모드**: DEV (스카팀·루나팀 코드 직접 수정 금지 — 점검·알림·패치 티켓까지만)

> 클로드팀은 시스템을 "돌아가게" 유지하는 팀이다.
> 다른 팀의 코드를 직접 고치지 않고, 이상을 감지하고 알리며 패치 티켓을 생성한다.

> **수집 금지**: BTC/ETH 가격, Fear&Greed Index → 루나팀 전담

**처리 흐름**

```
데이터 수집 (fetcher.js)
    ↓
Claude Sonnet 분석 (analyzer.js)
    ↓
docs/auto_dev/PATCH_REQUEST.md 생성 (patcher.js)  →  Claude Code RC 세션이 자동 감지
    ↓
텔레그램 주간 리포트 발송 (reporter.js)
```

**패치 우선순위**

| urgency | 내용 | 처리 |
|---------|------|------|
| `critical` | 보안 취약점·서비스 중단 위험 | 즉시 조치 |
| `high` | API 호환성 변경·주요 의존성 업데이트 | 사용자 확인 후 처리 |
| `medium` | 성능 개선·비필수 업데이트 | 자동 처리 |
| `low` | 기술 트렌드 적용·리팩토링 | 참고만 |

## 2. 스카팀

**역할**: 스터디카페 예약·운영 관리 전담
**위치**: `bots/reservation/`
**모드**: OPS (실서비스 운영 중)

> 스카팀은 네이버 스마트플레이스 예약과 픽코 키오스크를 실시간으로 동기화한다.
> 고객이 네이버에서 예약하면 자동으로 픽코에 등록되고, 키오스크 직접 예약은 네이버를 자동 차단한다.

## 부록: 팀 간 경계 규칙

```
클로드팀          스카팀               루나팀
  │               │                    │
  │  점검·알림     │  예약 자동화       │  자동매매
  │  패치 티켓     │  스터디카페 운영   │  3개 시장
  │               │                    │
  └── 다른 팀 DB 직접 접근 금지 ────────┘
      (team-bus.js 통해서만 통신)
```

| 규칙 | 내용 |
|------|------|
| 클로드팀 → 스카팀·루나팀 | 코드 직접 수정 금지. 점검·알림·티켓까지만. |
| 스카팀 ↔ 루나팀 | 독립 운영. 공유 데이터 없음. |
| 루나팀 → 클로드팀 | team-bus를 통한 이벤트 발행만 허용. |
| Gemini API | 스카팀 전용. 루나팀 사용 불가. |
| Groq API | 루나팀 전용 (9개 키 라운드로빈). |
