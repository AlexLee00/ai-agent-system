### 2-1. 기술분석가 테스트

**목표**: RSI/MACD/볼린저/이평 → 매수/매도/홀드 신호 정확도

| 항목 | 측정 방법 |
|------|----------|
| **속도** | TTFT + 총 응답 완료 시간 측정 |
| **출력 형식 준수** | JSON 파싱 성공률 (10회 반복) |
| **신호 일관성** | 동일 입력 5회 반복 → 신호 변동율 |
| **근거 품질** | 지표 해석의 정확도·구체성 (0~5점 수동 채점) |

**테스트 모델**: groq/llama-3.3-70b vs gemini-2.0-flash vs claude-haiku-4-5

---

### 2-2. 감성분석가 테스트

**목표**: 커뮤니티 텍스트 → 감성 점수 정확도

**샘플 입력 텍스트**:
```
"비트 존버중.. RSI 40 아래면 저점이지 않나요?"
"고점에서 산 코인이 반토막.. 손절해야하나"
"BTC ETF 유입 지속 중. 기관이 모으는 것 같음"
```

| 항목 | 측정 방법 |
|------|----------|
| **속도** | TTFT 측정 |
| **감성 분류 정확도** | 사람이 라벨링한 50개 샘플과 비교 |
| **장문 처리** | 5,000자 커뮤니티 글 처리 시 품질 |
| **한국어 이해도** | 코인판/DCInside 스타일 텍스트 처리 |

**테스트 모델**: gemini-2.0-flash vs groq/llama-4-scout vs claude-haiku-4-5

---

### 2-3. 온체인·뉴스분석가 테스트

**목표**: 단순 데이터 파싱 + 요약 → 속도 최우선

| 항목 | 측정 방법 |
|------|----------|
| **속도** | 10회 평균 TTFT |
| **구조화 출력** | JSON 파싱 성공률 |
| **요약 품질** | 핵심 정보 누락 여부 |

**테스트 모델**: groq/llama-3.1-8b vs groq/llama-4-scout (속도 비교 위주)

---

### 2-4. 리서처 (강세 ↔ 약세 토론) 테스트

**목표**: 4개 분석 보고서 → 매수/매도 근거 논리적 도출

| 항목 | 측정 방법 |
|------|----------|
| **논리 일관성** | 근거-결론 일치도 (0~5점) |
| **반대 의견 수용** | 약세 근거를 인식하는지 |
| **구체성** | 목표가·손절가 수치 포함 여부 |
| **속도** | 토론 1라운드 완료 시간 |

**테스트 모델**: claude-haiku-4-5 vs gemini-2.0-flash vs groq/llama-4-scout

---

### 2-5. 리스크 매니저 테스트

**목표**: 포지션 크기·리스크 수치 → 정확한 승인/수정/거부

**테스트 시나리오**:
- 시나리오 A: 총자산 12% 포지션 요청 → 10% 수정 권고해야 함
- 시나리오 B: 일손실 -2.8% 상태에서 신규 진입 요청 → 거부해야 함
- 시나리오 C: 3연속 손실 후 진입 요청 → 24시간 중지 권고해야 함

| 항목 | 측정 방법 |
|------|----------|
| **규칙 준수율** | 3가지 시나리오 정답률 |
| **수치 정확성** | 수정 포지션 크기 계산 정확도 |
| **응답 속도** | TTFT (빠를수록 좋음) |

**테스트 모델**: claude-haiku-4-5 vs groq/llama-3.3-70b

### 5분 주기 운영 시 일일 호출 수

| 에이전트 | 모델 | 일 호출 | 무료 한도 | 잔여율 |
|---------|------|---------|---------|--------|

## 4. 멀티계정 전략

### Groq 계정 로테이션

```javascript
// lib/groq-pool.js
const groqAccounts = [
  { apiKey: process.env.GROQ_KEY_1, dailyUsed: 0 },
  { apiKey: process.env.GROQ_KEY_2, dailyUsed: 0 },
  { apiKey: process.env.GROQ_KEY_3, dailyUsed: 0 }, // 선택적
];

function getNextGroqKey(model) {
  const limit = model.includes('70b') ? 1000 : 14400;
  return groqAccounts
    .filter(a => a.dailyUsed < limit * 0.8)  // 80% 도달 전 전환
    .sort((a, b) => a.dailyUsed - b.dailyUsed)[0];
}
```

### 계정별 한도 확장

| 계정 수 | llama-3.3-70b 일일 한도 | llama-3.1-8b 일일 한도 |
|--------|---------------------|-------------------|
| 1개 | 1,000 calls | 14,400 calls |
| 2개 | 2,000 calls | 28,800 calls |
| 3개 | 3,000 calls | 43,200 calls |

### Gemini 계정 로테이션

| 계정 | 용도 | 상태 |
|------|------|------|
| ***REMOVED*** | 현재 스카봇 primary | ✅ 사용 중 |
| (추가 계정 1) | 감성분석가 전용 | 🔄 생성 예정 |

### 수익 구간별 모델 업그레이드 로드맵

| 월 수익 구간 | 추가 API 예산 | 업그레이드 대상 | 효과 |
|------------|-------------|--------------|------|
| $0 ~ 초기 | $0 (현재) | 무료 모델 최대화 | 기본 운영 |
| 월 $100+ | +$5~10/월 | 리서처 → claude-sonnet | 토론 품질 30%↑ |
| 월 $500+ | +$20~30/월 | 분석가 → claude-haiku 전환 | 전체 품질↑, 속도↑ |
| 월 $2,000+ | +$50~100/월 | 전략 분석 → claude-opus 추가 | 심층 분석 가능 |

**역할별 업그레이드 우선순위**:
1. **리서처·리스크매니저** → 투자 결정 직결, 품질이 수익에 직접 영향
2. **기술분석가** → 신호 정확도 향상
3. **감성·뉴스분석가** → 현재 무료 모델도 충분한 경우 많음
