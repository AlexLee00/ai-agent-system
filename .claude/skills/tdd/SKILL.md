# /tdd — 테스트 주도 개발

## 절차 (RED → GREEN → REFACTOR)

### RED (실패 테스트 작성)
1. 구현할 기능의 **기대 동작**을 테스트로 먼저 작성
2. 테스트 실행 → 반드시 **실패** 확인 (아직 구현 없으므로)
3. 테스트가 "올바르게 실패"하는지 확인 (엉뚱한 이유로 실패하면 안 됨)

### GREEN (최소 구현)
1. 테스트를 **통과시키는 최소한의 코드**만 작성
2. 완벽한 코드가 아니어도 OK — 테스트 통과가 목표
3. 테스트 실행 → **통과** 확인

### REFACTOR (개선)
1. 테스트가 통과하는 상태에서 코드 정리
2. 중복 제거, 네이밍 개선, 구조 정리
3. 리팩터링 후 테스트 재실행 → 여전히 **통과** 확인

## 팀 제이 테스트 규칙

- 테스트 프레임워크: Node.js 기본 `node --test` 또는 프로젝트 내 test 스크립트
- 테스트 위치: 해당 팀 디렉토리 내 `tests/` 또는 `__tests__/`
- DB 테스트: 반드시 DEV 환경에서만 (MODE=dev 확인)
- OPS DB 직접 테스트 금지 — SSH 터널 + DEV 모드만 허용
- 테스트 후 반드시 `node --check [파일]`로 문법 검사

## 테스트 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| 문법 | node --check | 파싱 에러 없음 |
| 소프트 | 함수 단위 import + 호출 | export 확인, 반환값 검증 |
| 하드 | 실제 DB/API 연동 | OPS SSH 터널 경유 E2E |
