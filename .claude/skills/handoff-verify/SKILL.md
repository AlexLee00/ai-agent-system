# /handoff-verify — 독립 컨텍스트 이중 검증

## 원칙

> "의사가 처방하고, 약사가 다시 확인하는 이중 안전장치"
> 같은 컨텍스트에서 작성+검증하면 같은 실수를 반복할 수 있다.
> 새 컨텍스트(fork)에서 독립적으로 검증한다.

## 절차

1. **의도 기록**: 이 변경의 목적과 기대 동작을 명시
2. **독립 검증 시작**: 새 컨텍스트(fork)에서 아래 체크리스트 실행
3. **결과 보고**: PASS/FAIL + 발견 사항

## 검증 체크리스트

### 빌드 검증
- [ ] `node --check` 모든 변경 파일 통과
- [ ] import/require 경로가 실제 파일과 일치
- [ ] 의존성 패키지가 package.json에 존재

### 기능 검증
- [ ] 변경된 함수의 입출력이 기대와 일치
- [ ] 엣지 케이스 (null, undefined, 빈 배열) 처리
- [ ] 에러 핸들링이 throw 대신 null 반환 패턴

### 안전 검증
- [ ] OPS 환경 직접 수정 코드 없음
- [ ] secrets/API키 하드코딩 없음
- [ ] 실투자 보호 규칙 준수 (TP/SL 등)
- [ ] DEV/OPS 환경 분기 올바름

### 정합성 검증
- [ ] DB 쿼리 결과가 기존 데이터와 호환
- [ ] 기존 API 인터페이스 유지 (breaking change 없음)
- [ ] 다른 팀 코드에 영향 없음
