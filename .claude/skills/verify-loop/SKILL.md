# /verify-loop — 자동 검증 재시도

## 절차

### Loop (최대 3회 반복)

```
Round 1:
  1. node --check [변경 파일] → 문법 검사
  2. 테스트 실행 (있다면)
  3. 성공 → 완료!
  4. 실패 → 에러 분석 → 자동 수정 → Round 2

Round 2:
  1. 수정된 코드 재검증
  2. 성공 → 완료!
  3. 실패 → 에러 분석 → 자동 수정 → Round 3

Round 3 (최종):
  1. 수정된 코드 재검증
  2. 성공 → 완료!
  3. 실패 → 마스터에게 보고 (자동 수정 한계)
```

## 자동 수정 범위 (안전한 것만)

### 자동 수정 가능
- 문법 에러 (누락 괄호, 세미콜론, import 경로)
- 타입 불일치 (명확한 경우)
- 누락 변수 선언
- 테스트 assertion 값 오류 (기대값 업데이트)

### 자동 수정 금지 (마스터 보고)
- 비즈니스 로직 변경
- DB 스키마 변경
- API 인터페이스 변경
- 실투자 관련 코드
- 여러 파일에 걸친 구조적 변경

## 팀 제이 규칙

- DEV 환경에서만 실행
- OPS 파일 직접 수정 절대 금지
- 3회 실패 시 반드시 중단 + 텔레그램 보고
- 각 Round의 변경 사항을 로그로 기록
